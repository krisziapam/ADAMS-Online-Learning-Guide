ADAMS System Overview

ADAMS is a database-driven admission document management system. It manages admission documents and applicant/student records using a relational database.

Defense line: ADAMS helps the office record, search, monitor, print, and archive admission records through a database-driven system.

Final User Roles

Role / Record TypeMeaning
SuperadminManages system-wide accounts, settings, categories, requirements, logs, and reports.
StaffHandles searching, updating, uploading, monitoring, printing, and archiving.
Student/ApplicantAdmission record only, not a final login role.

Main ADAMS Features

  • Record applicant/student information.
  • Manage student categories and requirement checklists.
  • Upload and monitor admission documents.
  • Search, print, verify, and archive records.
  • Track users, logs, print history, password resets, and archived students.

ERD, EERD, Symbols, Shapes, Cardinalities

ERD means Entity Relationship Diagram. It shows database entities/tables, attributes, keys, and relationships.

EERD means Enhanced Entity Relationship Diagram. It extends ERD by showing enhanced concepts like specialization and generalization.

Defense line: In ADAMS, USER is specialized into SUPERADMIN_USER and STAFF_USER only. Students/applicants are records, not final login users.

Normalization: UNF → 1NF → 2NF → 3NF

Main point: For normalization, explain database/table transformation, not only system UI.
StageDatabase ViewWhat ChangedConcern Fixed
UNFOne wide monitoring tableStudent details + category + multiple requirement/status/file columns in one row.Repeating groups and redundancy.
1NFOne requirement per rowRequirement1/Requirement2 columns become multiple rows.Atomic values.
2NFCore tables separatedstudents, student_categories, requirement_types, category_requirements, student_requirements.Less duplication.
3NFSupport tables separatedusers, documents, activity_log, print_logs, password_reset_tokens, archived_students.Security, audit trail, reporting.
One-minute answer: Our normalization started from a manual logbook-style admission record. In UNF, one row contained student details, category, and repeated requirement columns. In 1NF, each requirement became a separate row. In 2NF, we separated core data by purpose. In 3NF, we separated operational data such as users, logs, print logs, password reset tokens, documents, and archived students.

SQL Lessons

Quick Start

USE doc_admission_db;
SHOW TABLES;

DESCRIBE students;
DESCRIBE student_requirements;
DESCRIBE documents;

Basic SELECT

SELECT *
FROM students;

SELECT user_id, username, full_name, role, is_active
FROM users;

SELECT *
FROM students
WHERE last_name LIKE '%Santos%';

SELECT student_id, last_name, first_name, email
FROM students
ORDER BY last_name ASC, first_name ASC;

COUNT and GROUP BY

SELECT COUNT(*) AS total_students
FROM students;

SELECT role, COUNT(*) AS total_users
FROM users
WHERE role IN ('superadmin','staff')
GROUP BY role;

JOIN

SELECT s.student_id, s.student_name, s.email, c.category_name
FROM students s
LEFT JOIN student_categories c
 ON c.category_id = s.category_id
ORDER BY s.student_name;

SELECT c.category_name, r.requirement_name, r.description
FROM category_requirements cr
JOIN student_categories c
 ON c.category_id = cr.category_id
JOIN requirement_types r
 ON r.requirement_id = cr.requirement_id
ORDER BY c.category_name, r.requirement_name;

Defense Q&A

What is the main focus of your project?

ADAMS manages admission documents and applicant/student records using a relational database.

Why is ADAMS considered a database project?

Its features depend on tables, keys, relationships, and SQL queries.

What is a database?

A database is an organized collection of related data that can be stored, searched, updated, and managed.

What is a DBMS?

A DBMS is software used to create, manage, secure, and retrieve records from a database.

What is a primary key?

A primary key uniquely identifies each record. In ADAMS, students.student_id identifies each student.

What is a foreign key?

A foreign key links one table to another. In ADAMS, students.category_id links to student_categories.category_id.

How did ADAMS apply 1NF?

By removing repeating groups and making each requirement a separate record.

How did ADAMS apply 2NF?

By separating student data, category data, requirement type data, category requirements, and student requirements.

How did ADAMS apply 3NF?

By separating users, activity logs, print logs, password reset tokens, documents, and archived records.

What is ERD?

An ERD shows tables, attributes, keys, and relationships.

What is EERD?

An EERD extends ERD with enhanced concepts like specialization and generalization.
